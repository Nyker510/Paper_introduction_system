注意

データの大幅な変更や，システムの変更を施す際には，事前にサーバ上のディレクトリ自体のバックアップを取っておくことをおすすめします．
本システムはDjangoとPython以外は，paperintro/だけで完結しているため，paperintro/さえ保存していれば復元が可能です．

環境構築

新たなサーバ上で本システムを構築する際の注意点を載せています．
本システムは2016年現在，Ubuntu16.04上のpython3.5.2とDjangoで運用しています．
djangoは2016年現在では，pipによる次のコマンドでインストールできます．
sudo pip3 install django
sudo pip3 install django-bootstrap-form
paperintroディレクトリ内で，次のコマンドを実行すれば，本システムを起動できます．
python3 manage.py runserver [IPアドレス]:[ポート]

データベース

論文・参考書のデータベースはSQLite3形式で，~/paperintro/db.sqlite3です．
http://IPアドレス:ポート/admin/には論文データを管理するためのページが準備されています．
SQLite3を直接操作することが難しい場合は，管理者ページから一括のデータ操作が可能となっています．

その他インタフェース

HTMLに関する設定は，~/paperintro/cms/templates/cms/*.htmlで変更してください．
SQLiteとのインタフェースとしてDjangoを利用しているため，データベースの構造・ソーティング機能，その他新しい機能の追加を行う際にはcms下のPythonプログラムを変更してください．
